Diagnostic Board

To run the COMX emulator with the Diagnostic Board select checkbox 'Diagboard' (highlighted in yellow). The choice box (highlighted in orange) can be set-to 'on' to start the diagnostic SW or to 'off' to start BASIC with the diagnostic board connected.

By default the following ROMs will be selected:

  • diag_pal1.bin, mapped to hex 0-FFF instead of BASIC ROM (if the diagnostic board is set to 'on'). BASIC ROM will be switched back on the first INP 1 instruction.
  • diag_pal2_v1.1.bin, mapped to hex C0000-D7FF. This v1.1 version will perform the extended ROM check on BASIC ROM file comx35.1.1.bin.

Next to this additional RAM will be selected from hex D800-DFFF.

The ROMs can be change by clicking 'Setup' (highlighted in blue above) which will open the 'Diag Board Settings' window:

The following alternative ROMs are available and can be specified for either the PAL or NTSC versions (highlighted in yellow):

  • diag_pal1.bin, diag_pal2.bin, with extended ROM check for the original COMX BASIC ROM (comx35.bin).
  • diag_ntsc1.bin, diag_ntsc2_v1.1.bin, NTSC version with ROM check for comx35.1.1.bin
  • diag_ntsc1.bin, diag_ntsc2.bin, NTSC version with ROM check for the original COMX BASIC ROM (comx35.bin).

The PAL or NTSC ROM set will be selected automatically when video mode is changed between PAL and NTSC (see also Video Mode, Setup and Log)

Additionally the following options can be selected in the setup window:

  • ROM checksum: on or off (highlighted in orange)
  • Factory unit: yes or no (highlighted in blue), when the factory unit is selected additional keyboard repeat and debounce tests are done as well as an IDEN test where IDEN is connected to bit 2 of INP 1.
  • Cassette cables: connected or disconnected (highlighted in purple)

Click Save All (highlighted in red) to save any changed diagnostic board settings.

The following PC keys are mapped:

  • S: Step
  • R: Repeat
  • A: Abort
