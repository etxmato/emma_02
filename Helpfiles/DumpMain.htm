Memory Dump

The Memory Dump tool shows 256 bytes of memory on the GUI both in hexadecimal format and as ASCII characters. Multiple memory options are available including CPU memory, video memory, memory type and profiler information.

To reach the Memory Dump tab, first select the 'Tools' (yellow ellipse) tab and then the 'Memory Dump' (orange ellipse) sub tab.

The default view will show the first 256 bytes of main CPU memory (yellow ellipse below showing content in bytes and orange ellipse showing ASCII) from the emulated computer. To show another part of memory type in any address value in the top left address field or use the spin buttons to step through memory (red ellipse).

Note that the text in red 'No emulation running' indicates that no emulator was started and as such the data shown is not active and changing address value will not have any effect.

To change any address value (yellow ellipse) select the value and just change it. Note both ROM and RAM values can be changed!

When running a COMX, TMC600 or Pecom emulation the actual CDP 1870 characters will be shown, other computers will show ASCII (orange ellipse)

Next to the main CPU memory other memory dump options are available by changing the 'Memory' choice button:

  • Type: memory types defined for the main CPU memory, see also 'Memory Type' chapter.
  • Profiler: see Profiler
  • 1870 Page: CDP 1870 Page memory
  • 1870 Character: CDP 1870 Character memory, for the Telmac TMC-600 Character ROM is shown and for both TMC-600 and Cidelsa only 6 bits are used.
  • 1870 Colour: CDP 1870 Colour RAM used by the Telmac TMC-600 and Cidelsa. The TMC-600 uses one nibble (4 bits) to define colour and blink mode of the character on the same location in page memory. the Cidelsa uses 3 bits to define the colour of the character and line in the same location in character memory.
  • 6845 Character: MC6845 Character ROM
  • 6847 Character: MC6847 Character RAM, both internal and external ROMs are shown; internal from hexadecimal address 0 to 1FF and external from 200 to 7FF.
  • 6847 Video RAM: The first half will show bits 0 to 7 and the second half bit 8 to 11.
  • TMS Video RAM: Only 16384 addresses will be viewable.
  • VT Video RAM: VT50/VT100 video RAM
  • 1862/1864 Colour: CDP 1862 or CDP 11864 colour RAM
  • 8275 Character: Intel 8275 character RAM
  • 8275 Video RAM: Intel 8275 video RAM
  • VIP Video Sequencer: VIP2K Membership video sequencer ROM
  • RTC RAM: Elf 2000 & Pico/Elf V2 RTC RAM

In all modes except 'Type' and 'Profiler', memory values are shown and can be changed per memory page of 256 bytes, in 'Type' mode memory types (ROM, RAM, Video RAM etc) is shown and can be changed.

The following additional features are available on windows and OS X, currently not implemented in the Linux version: After editing one byte the next address will be selected automatically. After the last byte of the page is changed the next page will be shown.

Note 1: when using the any of the video modes above only the used address will be shown, the number of addresses depends on the emulated computer and video settings.
Note 2: None of the settings in the Memory Dump tab are saved in the Emma 02 configuration, so at every start of the Emma 02 emulator these settings are set to default.
